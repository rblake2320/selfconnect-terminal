# Moved → docs/TERMINAL-TO-TERMINAL.md

This document was renamed. SelfConnect's core mechanism is **the terminal itself
as the communication medium** (keystroke injection: write text → wait → send
Enter separately → read the reply back), not generic agent-to-agent messaging.
The old "A2A" framing understated that, so the guide now leads with the
terminal-to-terminal protocol and treats MCP and the BPC/TSK file-mailbox /
WebSocket transports as supporting governed side doors.

**See [TERMINAL-TO-TERMINAL.md](./TERMINAL-TO-TERMINAL.md).**
